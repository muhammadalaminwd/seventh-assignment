Q.1: How does Javascript work?

Javascript is a high-level interpreted programming language. It makes HTML code dynamic, makes static pages to dynamic, and creates the website interactive. So you can call it a dynamic scripting language.

Javascript is a synchronous single-threaded language. It can execute one command or one line of code at a time. Javascript has an execution context. There are 2 components, Memory and Code. Memory component contains variables and functions as key, value pair. Code component is the thread of execution that hold executed code.

To run javascript code we need a web browser or and editor which has already installed Node Js. Every browser got their engine to run javascript codes. The most popular web browser Google Chrome got their V8 engine. This is how javascript works.

Q.3: Difference between local storage and session storage?

Context Local Storage Session Storage
1. Definition localStorage is a property that allows JavaScript sites and apps to save key-value pairs in a web browser with no expiration date. The sessionStorage object stores data only for a session. It means that the data stored in the sessionStorage will be deleted when the browser is closed.
2. Method localStorage has 4 methods. They are setItem(), getItem(), removeItem() and clear(). sessionStorage has 5 methods. They are getItem(key), setItem(key, value), removeItem(key), key(position), clear()
3. Capacity It has 5MB/10MB storage capacity. It has 5MB storage capacity.
4. Durability It stores data permanently but It can be deleted via Javascript or manually It stores data only for the duration of a session and it is automatically deleted when the browser is closed.
5. Readability The client can read it. The client also can read it.
